phase 1 core

Phase 1 的重点主线，
是养虾36计。

每一个 tab 未来都必须能点进去、能拓展、能沉淀成长期资产。第一阶段先把养虾36计做成主线内容模块，每一计都不是一句标题，而是一篇完整的方法论文章。它不是内容摆设，而是未来整个网站最重要的方法骨架。

为什么先做 36 计

  • 它最容易建立体系感
  • 最适合持续连载与品牌记忆
  • 最容易承接诊断、陪跑和方法论输出
  • 最适合作为 Phase 1 的核心内容工程

每一计统一框架

  • 5 Why：连追根因
  • What：定义这一计是什么
  • Detail What：展开细节
  • So What：说明意义
  • How：给做法与举一反三

沿用真实三十六计原名

名字完全跟现实三十六计一致，但解释全部贴合“养虾 / Agent 养成”语境。

不是硬套典故

不是为了古风，而是借用熟悉框架，降低记忆成本、提高传播性。

每一计都能进入详情

它们不是一个个标签，而是未来可以持续扩展、持续更新的独立内容单元。

首页只是入口

首页负责讲清世界观，真正的内容资产会沉淀到各个模块详情页里。

content architecture

这个网站不是单页海报。
而是一个持续扩张的内容系统。

首页只负责讲清价值和导流。真正的长期价值，会沉淀在一个个可以点进去、可以扩展、可以持续更新的内容模块里。Phase 1 先打穿 36 计，Phase 2 再把其他模块逐步做厚。

养虾36计

策略与方法主线。当前 Phase 1 的绝对重点。

虾24章经

原则、认知与修炼模块。未来承接长期心法与判断框架。

虾说

洞察栏目。解释为什么大多数人养虾养不大，建立品牌认知高地。

虾评

案例点评模块。帮用户看清自己的配置、workflow 和养法到底卡在哪里。

为什么每个 tab 都必须能进去

  • 不能只是首页展示名词
  • 必须能承载持续更新的内容
  • 未来要形成真正的知识资产与品牌资产

为什么网站内容必须足够

  • 高级感只能吸引第一眼
  • 真正让人停留和信任的是内容厚度
  • 有内容，才有转化、传播与长期沉淀的可能
growth stages

成长不是跳跃。
而是有阶段的升级。

看清自己在哪一层，比盲目加工具更重要。方向对了，进化才会开始加速。

Stage 01

会装

能装起来，但产出不稳定。

Stage 02

会跑

能执行任务，但仍依赖人工盯。

Stage 03

会用

能出结果，但还缺系统方法。

Stage 04

会养

知道如何让判断、协作和记忆持续变强。

Stage 05

会经营

把 Agent 当作真正的数字员工来运营。
