## 1. 北极星指标

### 🌟 北极星：周活跃回访用户数（Weekly Returning Visitors, WRV）

| 维度 | 说明 |
|------|------|
| **定义** | 一周内至少访问2次且停留>30s的独立用户数 |
| **计算** | Umami `returning visitors` × `avg duration > 30s` 过滤 |
| **为什么不是DAU** | 导入期日活太小（可能个位数），波动剧烈无意义；周粒度更稳定 |
| **为什么不是UV** | UV是虚荣指标——100个UV跳出率95%不如10个UV回访率50% |
| **为什么不是订阅数** | 订阅功能未上线，无法度量 |
| **核心逻辑** | 回访 = 内容有价值 + 用户有需求 = PMF的最早信号 |

### 北极星与产品阶段映射

| PLC阶段 | 北极星指标 | 切换条件 |
|---------|-----------|---------|
| **导入期（当前）** | 周回访用户数 WRV | WRV > 50 且连续4周 |
| 成长期 | 周活跃用户数 WAU | WAU > 500 且连续4周 |
| 成熟期 | 月活跃用户数 MAU | — |

> ⚠️ 北极星指标不是永久的。每个阶段关注一个，达标后升级。

## 2. AARRR 漏斗指标体系

### 2.1 Acquisition（获取）

> 用户从哪里来？哪个渠道效率最高？

| 指标 | 定义 | 数据源 | 导入期目标 |
|------|------|--------|-----------|
| **总UV** | 周独立访客数 | Umami | Week 1: >30 / Month 1: >500 |
| **渠道UV分布** | 按来源拆分：Direct / Organic / Social / Referral | Umami referrer + UTM | 有≥3个渠道贡献UV |
| **渠道获客成本（CAC）** | 各渠道投入时间 ÷ 带来UV | 手动记录 | 导入期为0（纯免费） |
| **GSC曝光量** | Google搜索结果展示次数 | GSC | Week 2: >100次展示 |
| **GSC点击率（CTR）** | 搜索点击 ÷ 展示 | GSC | >3%（搜索行业基准） |
| **GSC索引页数** | 已被Google收录的页面数 | GSC | Month 1: >50页（目标69页全收录） |

**渠道归因规则**：
```
UTM参数标准：
- Twitter:  ?utm_source=twitter&utm_medium=social&utm_campaign=launch
- 即刻:     ?utm_source=jike&utm_medium=social&utm_campaign=launch
- V2EX:     ?utm_source=v2ex&utm_medium=community&utm_campaign=launch
- Telegram:  ?utm_source=telegram&utm_medium=social&utm_campaign=channel
- 少数派:    ?utm_source=sspai&utm_medium=content&utm_campaign=article
- ProductHunt: ?utm_source=producthunt&utm_medium=launch&utm_campaign=ph1
- 无UTM+无referrer: 归类为 Direct
- 有referrer无UTM: 归类为 Referral，记录referrer域名
```

### 2.2 Activation（激活）

> 用户来了之后，有没有产生有效互动？

| 指标 | 定义 | 数据源 | 导入期目标 |
|------|------|--------|-----------|
| **激活率** | 激活用户 ÷ 总UV | Umami事件 | >30% |
| **首次有效互动** | 满足以下任一条件即视为「激活」 | Umami | — |
| ↳ 阅读深度 | 单篇文章滚动到底部（scroll_depth = 100%） | 自定义事件 | — |
| ↳ 多页浏览 | 单次会话浏览 ≥3 个不同页面 | Umami pageview | — |
| ↳ 停留时长 | 单次会话停留 ≥2 分钟 | Umami duration | — |
| ↳ 互动行为 | 点击AI提示词复制按钮 / 进入游戏页 / 查看BN-Alpha报告 | 自定义事件 | — |
| **跳出率** | 仅看1页即离开的会话占比 | Umami | <65%（内容站基准） |
| **平均停留时长** | 全站平均会话时长 | Umami | >1.5分钟 |
| **页面深度** | 平均每会话浏览页数 | Umami | >2.0页 |

**激活定义的底层逻辑**：
```
激活 = 用户从「路过」变成「消费」
     = 证明内容匹配了用户需求
     = AARRR漏斗中最关键的转化点

不算激活的行为：
- 只看首页就走（bounce）
- 停留<10秒（机器人/误点）
- 只看了导航页没进任何文章
```

### 2.3 Retention（留存）

> 用户走了还会回来吗？

| 指标 | 定义 | 数据源 | 导入期目标 |
|------|------|--------|-----------|
| **D1留存率** | 次日回访用户 ÷ 当日新用户 | Umami | >5%（内容站基准） |
| **D7留存率** | 7日内回访 ÷ 首访用户 | Umami | >10% |
| **D30留存率** | 30日内回访 ÷ 首访用户 | Umami | >15%（SC-001目标） |
| **回访频率** | 回访用户的平均访问间隔（天） | Umami | <7天 |
| **回访深度** | 回访用户平均浏览页数（vs新用户对比） | Umami | 回访 > 新用户 1.5x |
| **内容消耗进度** | 回访用户累计阅读的36计文章数 | 自定义事件 | 平均>5篇/用户 |

**留存分群**：
```
新用户（New）: 首次访问
回头客（Returning）: 2-3次访问
活跃读者（Active）: ≥4次访问
忠实用户（Loyal）: ≥8次访问 + 累计阅读≥10篇
```

### 2.5 Referral（推荐）

> 用户愿意推荐给别人吗？

| 指标 | 定义 | 数据源 | 导入期目标 |
|------|------|--------|-----------|
| **分享点击率** | 分享按钮点击 ÷ 文章PV | 自定义事件 | >2%（需先部署分享按钮） |
| **外链数** | 指向singclaw.xyz的外部链接数 | GSC + Ahrefs | Month 1: ≥3 |
| **社交提及数** | Twitter/即刻等提及SingClaw的帖子数（非自己发的） | 手动监控 | Month 1: ≥1 |
| **K系数** | 每个用户平均带来的新用户数 | 计算：邀请发送×转化率 | >0.3（导入期） |
| **NPS** | 净推荐值（需调研） | 用户访谈 | 首批用户中NPS>30 |

**K系数计算**：
```
K = 分享用户占比 × 每次分享触达人数 × 触达→访问转化率
  = share_rate × reach × click_rate

示例（导入期预估）：
K = 5% × 50 × 2% = 0.05（远低于1，无法自增长）
→ 导入期的核心任务是把K提到>0.3，为病毒式引擎打基础
```

### 6.3 自定义事件埋点清单

#### A. 内容消费类事件

| 事件名 | 触发条件 | 参数 | 用途 |
|--------|---------|------|------|
| `article_read` | 文章页停留>30s | `{title, category, slug}` | 统计实际阅读（非扫一眼） |
| `scroll_depth` | 滚动到25%/50%/75%/100% | `{depth, page}` | 判断内容消费深度 |
| `article_complete` | 滚动到文章底部（100%） | `{title, category, duration}` | 完读率计算 |
| `series_progress` | 用户在36计系列中累计阅读篇数 | `{count, latest_article}` | 系列内容粘性 |

**埋点代码示例（article_read）**：
```javascript
// 在每篇文章页底部添加
let readTimer = null;
document.addEventListener('DOMContentLoaded', () => {
  readTimer = setTimeout(() => {
    umami.track('article_read', {
      title: document.title,
      category: document.querySelector('meta[name="category"]')?.content || 'unknown',
      slug: window.location.pathname
    });
  }, 30000); // 30秒后触发
});
window.addEventListener('beforeunload', () => clearTimeout(readTimer));
```

#### B. 互动行为类事件

| 事件名 | 触发条件 | 参数 | 用途 |
|--------|---------|------|------|
| `copy_prompt` | 点击AI提示词复制按钮 | `{prompt_id, article}` | 提示词库使用率 |
| `click_internal_link` | 点击文章内的交叉引用链接 | `{from_page, to_page}` | 内链网络效果 |
| `click_external_link` | 点击外部链接（如参考资料） | `{url, page}` | 用户流出追踪 |
| `click_nav` | 点击导航菜单项 | `{menu_item}` | 导航使用模式 |
| `search_query` | 站内搜索（如有） | `{query}` | 用户需求洞察 |

**埋点代码示例（copy_prompt）**：
```javascript
// 绑定所有复制按钮
document.querySelectorAll('.copy-prompt-btn').forEach(btn => {

      prompt_id: btn.dataset.promptId,
      article: document.title
    });
  });
});
```

## 9. 数据异常检测规则

### 自动检测逻辑

```
异常判定 = 当前值偏离7日移动平均 > 2个标准差

具体规则：
1. UV骤降>50% (环比日)    → 🔴 网站可达性检查
2. 跳出率骤升>20个百分点   → 🔴 着陆页质量检查
3. 某渠道UV归零           → 🟡 渠道链接/帖子检查
4. 停留时长骤降>50%        → 🟡 内容质量/加载速度检查
5. GSC索引数下降           → 🔴 立即检查是否被deindex
6. Umami数据中断>1小时     → 🔴 统计服务修复
```

### 异常响应SLA

| 级别 | 响应时间 | 动作 |
|------|---------|------|
| 🔴 红色 | <2小时 | 立即诊断+修复+通知明哥 |
| 🟡 黄色 | <24小时 | 记录+分析+下次周报重点提及 |
| ℹ️ 信息 | 下次周报 | 记录观察 |
