## 三、Kano 需求分析

### 3.1 基本需求（Must-Be）— 没有就走人

| 需求 | 说明 | 当前状态 |
|------|------|---------|
| **中文界面** | 目标用户80%+中文母语 | ✅ 已实现 |
| **移动端适配** | 碎片时间=手机端 | ⚠️ 需验证响应式 |
| **免费上手** | 任何预付费=流失90%用户 | ✅ F2P设计 |
| **离线/挂机收益** | Idle核心=不在线也能涨 | ⚠️ 需核心循环验证 |
| **资产安全** | 不能Rug，合约可验证 | ⚠️ 需部署测试网 |

### 3.2 期望需求（One-Dimensional）— 做得越好满意度越高

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **养成深度** | 多虾种+进化+稀有度 | P1 |
| **收益可感知** | 每日/每周收益Dashboard | P1 |
| **社交功能** | 看好友虾塘+排行榜 | P2 |
| **任务系统** | 每日/每周任务引导行为 | P2 |
| **Gas优化** | Base低Gas但仍需抽象 | P1 |

### 3.3 兴奋需求（Attractive）— 有了会惊喜

| 需求 | 说明 | 差异化价值 |
|------|------|-----------|
| **虾种NFT** | 稀有虾可交易，收藏属性 | 独特IP溢价 |
| **季节/天气系统** | 不同季节产出不同资源 | 增加回访动力 |
| **虾塘PvP** | 我的虾vs你的虾比赛 | 社交病毒传播 |
| **跨游戏彩蛋** | 与SingClaw其他板块联动 | 生态网络效应 |
| **零钱包体验** | 邮箱登录即玩，后台托管钱包 | 消灭Web3最大门槛 |

### 3.4 Kano优先级矩阵

```
满意度 ↑
  │        ╱ 兴奋需求
  │       ╱  (虾种NFT/PvP/零钱包)
  │      ╱
  │     ╱
  │────╱───── 期望需求 (养成深度/收益可见/社交)
  │  ╱
  │ ╱
  │╱_________ 基本需求 (中文/移动端/免费/离线/安全)
  └──────────────────────→ 实现程度
```

**结论：当前阶段应优先确保5个基本需求100%满足，然后集中攻P1期望需求（养成深度+收益可感知+Gas抽象）。兴奋需求留给MVP后迭代。**

## 四、TAM/SAM/SOM 市场规模估算

### 4.1 数据基础

| 指标 | 数值 | 来源 |
|------|------|------|
| 全球GameFi市场规模(2024) | $16.3B | MRFR / SkyQuest |
| 全球GameFi市场规模(2025E) | $21B | MRFR |
| 2026E预测 | $20-30B | GameFi Times |
| 2035预测 | $257.9B (CAGR 28.5%) | MRFR |
| 全链日活钱包 | 4.8-7.3M dUAW | DappRadar 2025 |
| 中国/中文用户占比(估) | ~15-20% | 行业估算 |
| Idle/休闲类占GameFi比例(估) | ~8-12% | PlayToEarn品类分析 |
| Base链游戏用户(估) | ~50-100K dUAW | NFTPlazas/DappRadar |

### 4.2 TAM — 总可寻址市场

**定义**：全球所有可能玩区块链Idle/休闲游戏的用户

```
TAM = 全球GameFi市场 × Idle/休闲占比
    = $21B × 10%
    = ~$2.1B (2025)
```

按用户数：
```
TAM(用户) = 全链游戏dUAW × 休闲占比 × 30天
          = 5.5M × 10% × 30
          ≈ 16.5M MAU
```

### 4.3 SAM — 可服务市场

**定义**：中文用户 + Base链可触达 + Idle品类

```
SAM = TAM × 中文用户占比 × Base可触达比例
    = $2.1B × 18% × 15%
    = ~$56.7M
```

按用户数：
```
SAM(用户) = 16.5M × 18% × 15%
          ≈ 445K MAU
```

### 4.4 SOM — 可获取市场（Year 1 目标）

**定义**：ShrimpFi第一年现实可获取的用户与收入

参考同级项目数据：
- FrenPet (Base, Idle): ~1-2K DAU
- Sunflower Land (Polygon, 农场): ~5-8K DAU
- GMonchain (Base, 社交): ~4K UAW

```
SOM(保守) = 1K DAU × 30天 = ~30K MAU
SOM(乐观) = 5K DAU × 30天 = ~150K MAU

SOM(收入-保守) = 30K MAU × $0.5 ARPU/月 = ~$15K/月 = ~$180K/年
SOM(收入-乐观) = 150K MAU × $1.0 ARPU/月 = ~$150K/月 = ~$1.8M/年
```

## 五、JTBD (Jobs-to-be-Done) 分析

### 5.1 核心Job

> **「当我在通勤/等待/无聊时，我想要一个不费脑但有收获感的链上游戏，这样我既能消磨时间，又能积累链上资产，还不用担心被割。」**

### 5.2 Job分解

| Job层级 | 描述 | 当前替代方案 | 替代方案不足 |
|---------|------|-------------|-------------|
| **功能性Job** | 碎片时间获得链上收益 | Notcoin(点击)/DeFi质押 | 点击太无聊/DeFi太复杂 |
| **情感性Job** | 养成收集的满足感 | 旅行青蛙/动物餐厅 | 无链上价值/无收益 |
| **社交性Job** | 有可展示的成就 | 社交媒体晒图 | 游戏无社交货币 |

### 5.3 Hiring Criteria（用户选择ShrimpFi的决策因素）

1. **10秒能上手** — 不用看教程就知道怎么玩
2. **不花钱就能玩** — F2P是底线
3. **中文** — 看得懂所有按钮和规则
4. **离线有收益** — 关掉页面虾还在长
5. **不会被Rug** — 团队可信/合约透明/Base背书

## 六、关键结论与Gate建议

### 6.1 Go/No-Go 评估

| 维度 | 评分 | 理由 |
|------|------|------|
| 市场存在性 | ✅ 8/10 | GameFi $21B市场，Idle品类有验证(Pixels 1M DAU) |
| 竞争空白 | ✅ 9/10 | Base×Idle×中文=几乎无竞品 |
| 用户需求真实性 | ⚠️ 6/10 | 基于竞品类推，缺少一手VOC数据 |
| 团队执行力 | ⚠️ 5/10 | P3/P4阻塞于钱包私钥，核心循环未验证 |
| 时机 | ✅ 8/10 | Base生态爆发期(TVL $15B)，GameFi行业重建期 |

### 6.2 建议

**小洞建议：有条件Go，进入Concept阶段。**

条件如下：
1. ⚠️ **必须验证核心Idle循环** — 部署测试网，让10个真实用户试玩48小时，验证留存
2. ⚠️ **必须采集一手VOC** — 在即刻/Twitter发问卷，最少收30份，验证Persona假设
3. ⚠️ **必须解决钱包私钥P0阻塞** — 没有链上部署，一切分析都是空中楼阁

> 📌 如果以上3个条件在Sprint #2内无法满足，建议在Gate Review时降级为Hold。
> 数据面前没有"我觉得用户会喜欢"。
